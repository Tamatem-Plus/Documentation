Initialization Prerequisites

Before starting you should set your deep linking and prepare the configuration file for the SDK.

Setting Deep linking.

Start by selecting your project file, select the target, go to the info tab, and unfold the URL Types section:

IDENTIFIER

The identifier distinguishes your app from others in case multiple apps declare support for the same scheme. Using your app’s bundle ID as a prefix is best practice, but it doesn’t prevent any other apps from registering the same scheme. If multiple apps register the same scheme, the app the system targets will be undefined. You should consider using Universal Links instead if uniqueness becomes essential.

URL SCHEMES

Specifies the URL scheme to use for opening links in your app.In this case, we’ve configured our app to handle URLs starting with tamatem-testkuwait://You will get yours from us.

Implement Info plist configuration file.

Tamatem SDK depends on some predefined keys to get important values to run the SDK.In your project root directory create a new plist file and name it “TamatemConfiguration”and fill it with the following keys:

  • Client ID is your game ID into the Tamatem Plus system, which we will provide you with once integration starts.
  • Redirect URI: The link is the game link where the user will be redirected back after finishing the authorization process to continue the journey. For a better user experience, we would recommend you use the following format (companyName-gameName://companyName).
  • Base URL: To start testing the SDK, please use our staging environment URL "https://stg-be.tamatemplus.com/".
