How to test the connection with Tamatem ID?

Our SDK was built to let you focus on designing an enjoyable experience for your users! Please follow these steps to get started with our example scene:

  1. Import the Tamatem Plus SDK Unity Package. You can do that by one of the following methods:
    1. While having your Unity project open, double-click on the Unity package.
    2. Drag the Unity package and drop it into your Unity project.
    3. In your Unity project, from the menu bar, go to Assets → Import Package → Custom Package, then choose our Unity package.
  2. We have the Newtonsoft.Json library included in the package. If you already have it in your project, please exclude it when you import the package.
  3. Enter your client ID and redirect URI. From the menu bar, go to Tamatem → Connect Settings, and enter your correct information.
  4. Open our example scene. In the “Project” tab, Go to TamatemConnect → Example → ExampleScenes and double-click the ConnectExample scene.
  5. Add the ConnectExample scene to “Scenes in Build” by going to File → Build Settings → Add Open Scenes.
  • Client ID is your game id into Tamatem Plus system, we will provide you with once integration starts.
  • Redirect URI: The link is the game link where the user will be redirected back after finishing the authorization process to continue the journey. For a better user experience, we would recommend you use the following format (companynamegamename://companyname).

Building the example scene for Android:

  1. From “Build Settings”, under “Platform”, choose Android and click on “Switch Platform”.
  2. Go to Player Settings → Other Settings and make sure you change the “Scripting Backend” to “IL2CPP” and include both the “ARMv7” and “ARM64” Target Architectures.
  3. Go to Player Settings → Publishing Settings and check the Custom Main Manifest option. This will generate a custom manifest file that you can find in Assets → Plugins → Android. Open this file and add a new , add the following four lines inside, and save the file: 
    4. <intent-filter>
    <action android:name ="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name ="android.intent.category.BROWSABLE" />
    <data android:scheme="companyname-gamename" android:host="companyname"/>
</intent-filter>
    Do not forget to replace the “companyname-gamename” and the “companyname” with the actual info that you used in the redirect URI. Without this you will not be able to complete the connection process.
  4. Click on Build!

Building the example scene for iOS:

  1. From “Build Settings”, under “Platform”, choose Android and click on “Switch Platform”.
  2. Go to Player Settings → Other Settings and make sure you change the “Scripting Backend” to “IL2CPP” and include both the “ARMv7” and “ARM64” Target Architectures.
  3. Click on Build and create a new folder for the exported Xcode project.
  4. Open the exported Xcode project.
  5. Under “PROJECT”, go to “Build Settings”, search for “Enable Bitcode” and set it to “No”.
  6. Under “TARGETS”, click on “Unity-iPhone”, go to “Signing & Capabilities”, enable “Automatically manage signing”, and choose your Team.
  7. Repeat step no. 5 for both “Unity-iPhone Tests” and “UnityFramework”.
  8. Connect your iOS device and enable the “Developer Mode”.
  9. Click on Play in Xcode.
